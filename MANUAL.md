# MedSafety Link 運用マニュアル

最終更新: 2026-05-05

## 1. MedSafety Linkとは

MedSafety Linkは、医療機関がLINE公式アカウントとGoogleスプレッドシートを使って、患者さんへの連絡、安否確認、返信確認を行うためのシステムです。

主な用途は次の通りです。

- 災害時の安否確認
- 通常診療時の連絡受付
- 患者さんへの一斉送信
- 未回答者への再送
- 個別メッセージ送信
- 回答履歴、送信履歴、対応状況の確認
- 足病変などの写真受信とGoogle Drive保存
- 複数施設ごとのLINE設定、Google設定の管理

実運用ではRender版を使用します。

```text
https://medsafety-link.onrender.com
```

LINE Developersに設定するWebhook URLは次です。

```text
https://medsafety-link.onrender.com/callback
```

## 2. 用意するもの

初回設定では、次の4つを用意します。

| 用意するもの | 用途 | リンク |
| --- | --- | --- |
| LINE公式アカウント | 患者さんへの送信、患者さんからの返信受信 | https://manager.line.biz/ |
| LINE DevelopersのMessaging APIチャネル | チャネルアクセストークン、Webhook設定 | https://developers.line.biz/console/ |
| Googleスプレッドシート | 患者情報、回答、送信履歴の保存 | https://docs.google.com/spreadsheets/ |
| Google Driveフォルダ | 足病変などの写真保存 | https://drive.google.com/ |
| Google CloudサービスアカウントJSON | MedSafety LinkがGoogleスプレッドシートへ読み書きするための認証 | https://console.cloud.google.com/ |

## 3. ログインと新規施設登録

ログイン画面:

```text
https://medsafety-link.onrender.com/login
```

既存施設は、施設IDとパスワードでログインします。

新しい施設を追加する場合は、ログイン画面の「新規施設登録」を押します。次の項目を入力します。

- 施設名
- 診療科名
- 施設ID
- パスワード
- パスワード確認

登録後、その施設IDで自動ログインし、設定画面へ進みます。

注意: 新規登録ボタンは公開ログイン画面に表示されます。登録を制限したい場合は、今後「招待コード」方式を追加する運用が考えられます。

## 4. 管理画面メニュー

ログイン後、上部に現在のモードとメニューが表示されます。

- ダッシュボード
- 患者登録
- 患者一覧
- 回答履歴
- 送信履歴
- 一斉送信
- 未回答再送
- モード切替
- 設定
- Google連携手順
- 施設ID管理
- ログアウト

「施設ID管理」は、システム管理者として許可された施設IDでログインした場合だけ表示されます。現在の運用では、`kumamoto_chuo` でログインした場合に表示されます。

## 5. 初回設定の流れ

新規施設登録後は、設定画面で次の順に進めます。

1. 施設名、診療科、電話番号、ログインパスワードを確認します。
2. LINEチャネルアクセストークンを入力して保存します。
3. LINE接続診断を実行します。
4. LINE destination / Bot userIdが自動保存されたことを確認します。
5. Google CloudでサービスアカウントJSONを作成します。
6. Google Sheets APIを有効化します。
7. 設定画面でサービスアカウントJSONをアップロードして保存します。
8. 表示された共有先メールアドレスをGoogleスプレッドシートに編集者として共有します。
9. スプレッドシートIDを入力して保存します。
10. 足病変写真などを保存する場合は、Google DriveフォルダIDを入力して保存します。
11. Google接続診断を実行します。
12. Googleシート初期化を実行します。
13. 患者登録を行います。
14. LINE DevelopersでWebhook URLを設定します。
15. テスト患者で送信、返信、回答表示、画像受信を確認します。

## 6. 設定画面

設定画面では、ログイン中の施設だけに適用される設定を保存します。

### 6-1. 初回設定で用意するもの

設定画面の上部に、LINE公式アカウント、LINE Developers、Googleスプレッドシート、Google Cloudへのリンクを表示しています。初めて設定する施設では、この案内を見ながら準備してください。

### 6-2. 施設情報

入力項目:

- 施設名
- 診療科
- 電話番号
- ログインパスワード

電話番号は、通常モードで時間外に患者さんからメッセージが届いた時の自動返信に表示できます。

### 6-3. LINE設定

入力項目:

- チャネルアクセストークン
- LINE destination / Bot userId
- 管理者LINE ID

チャネルアクセストークンは、LINE Developersの対象チャネルで取得します。

```text
LINE Developers
対象チャネル
Messaging API
チャネルアクセストークン
```

保存後、「LINE接続診断」を押します。成功すると、LINE destination / Bot userIdが自動保存されます。

管理者LINE IDは、緊急回答が届いた時に通知を受け取る医療者のLINE user_idです。複数人へ通知する場合はカンマ区切りで入力します。

```text
Uxxxxxxx,Uyyyyyyy
```

### 6-4. メッセージ設定

施設ごとに標準文面を設定できます。

- 一斉送信の標準メッセージ
- 未回答再送の標準メッセージ
- 個別送信の標準メッセージ
- 個別送信の定型文3件

一斉送信、未回答再送、個別送信の画面でも、送信前に文面を編集できます。

### 6-5. 安否確認ボタン・回答コード

LINE上に表示するボタン名と、回答履歴に保存するコードを施設ごとに設定できます。

標準例:

| ボタン | 表示名 | 回答コード |
| --- | --- | --- |
| 1 | 無事 | SAFE |
| 2 | 体調不良 | SICK |
| 3 | 薬・インスリン不足 | INSULIN_OUT |
| 4 | 低血糖が心配 | HYPO |
| 5 | 至急連絡希望 | CALL |

LINE内部では1から5の値で送受信し、MedSafety Link側で施設設定に合わせて表示名と回答コードへ変換します。

### 6-6. Google設定

入力項目:

- サービスアカウントJSONファイル
- スプレッドシート共有先メールアドレス
- 施設専用スプレッドシートID

「スプレッドシート共有先メールアドレス」は、アップロードされたJSONの `client_email` を表示します。この欄は直接編集できません。変更したい場合は、新しいサービスアカウントJSONをアップロードして保存します。

設定画面下部の診断ボタン:

- LINE接続診断
- Webhook受信診断
- Google接続診断
- Googleシート初期化
- 設定保存先診断

設定保存先診断は、施設ID管理が見える管理者施設だけに表示されます。

## 7. Google連携手順

### 7-1. Google CloudでサービスアカウントJSONを作成する

Google Cloud Console:

```text
https://console.cloud.google.com/
```

手順:

1. 使用するGoogle Cloudプロジェクトを選択します。
2. 「IAMと管理」から「サービスアカウント」を開きます。
3. サービスアカウントを作成します。
4. 作成したサービスアカウントを開きます。
5. 「キー」から新しいキーを作成します。
6. キーの種類はJSONを選択します。
7. JSONファイルをダウンロードします。

### 7-2. Google Sheets APIを有効化する

Google Sheets API:

```text
https://console.cloud.google.com/apis/library/sheets.googleapis.com
```

対象プロジェクトでGoogle Sheets APIを有効化します。有効化直後は反映に数分かかることがあります。

足病変などの写真をGoogle Driveへ保存する場合は、同じGoogle CloudプロジェクトでGoogle Drive APIも有効化します。

Google Drive API:

```text
https://console.cloud.google.com/apis/library/drive.googleapis.com
```

### 7-3. JSONをMedSafety Linkへアップロードする

1. MedSafety Linkの「設定」を開きます。
2. 「サービスアカウントJSONファイル」でJSONを選択します。
3. 「保存」を押します。
4. 「スプレッドシート共有先メールアドレス」が表示されます。

### 7-4. スプレッドシートを共有する

1. 使用するGoogleスプレッドシートを開きます。
2. 右上の「共有」を押します。
3. MedSafety Linkに表示された共有先メールアドレスを追加します。
4. 権限を「編集者」にします。
5. 共有を完了します。

### 7-5. スプレッドシートIDを入力する

スプレッドシートURLの `/d/` と `/edit` の間の文字列がスプレッドシートIDです。

```text
https://docs.google.com/spreadsheets/d/ここがスプレッドシートID/edit
```

このIDを設定画面の「施設専用スプレッドシートID」に入力し、保存します。

### 7-6. 画像保存先Google DriveフォルダIDを入力する

足病変などの写真を受け取る運用では、Google Driveに画像保存用フォルダを作成します。

手順:

1. Google Driveで画像保存用フォルダを作成します。
2. フォルダを開きます。
3. URLの `/folders/` の後ろの文字列をコピーします。
4. MedSafety Linkの設定画面の「画像保存先Google DriveフォルダID」に貼り付けます。
5. そのDriveフォルダを、設定画面に表示されているスプレッドシート共有先メールアドレスへ編集者として共有します。

例:

```text
https://drive.google.com/drive/folders/ここがDriveフォルダID
```

DriveフォルダIDが未設定の場合、画像はGoogle Driveへ保存されず、LINE画像IDから表示を試みます。LINE側の保存期間や取得制限を考えると、実運用ではGoogle Drive保存を推奨します。

### 7-7. Google接続診断と初期化

1. 「Google接続診断」を押します。
2. 成功したら「Googleシート初期化」を押します。

初期化で確認、作成されるタブ:

- `patients`
- `pending_users`
- `responses`
- `sent_messages`
- `system_mode`

通常、既存データは削除せず、必要な見出し行を整えます。

## 8. LINE DevelopersとLINE公式アカウントの設定

LINE Developersで対象のMessaging APIチャネルを開きます。

確認項目:

- Webhook URL: `https://medsafety-link.onrender.com/callback`
- Webhookの利用: ON
- Webhook検証: 成功
- チャネルアクセストークン: MedSafety Linkの設定画面に保存済み

LINE Official Account Manager側の確認項目:

- 応答モード: Bot
- Webhook: ON
- 応答メッセージ: OFF
- AI応答メッセージ: OFF

患者さんが返信した時にLINE公式アカウント側の固定文が返る場合は、応答メッセージまたはAI応答メッセージがONの可能性があります。

## 9. 患者登録

患者登録画面では、患者情報とLINE user_idを紐付けます。

患者さんがLINE公式アカウントへメッセージを送ると、「登録待ちユーザー」に表示されます。

登録の流れ:

1. 患者さんにLINE公式アカウントを友だち追加してもらいます。
2. 患者さんに「テスト」などのメッセージを送ってもらいます。
3. 管理画面の「患者登録」を開きます。
4. 登録待ちユーザーのLINE user_idを確認します。
5. 患者ID、氏名、電話番号、LINE user_id、患者区分、メモを入力または紐付けます。
6. 保存します。

LINE user_idと患者情報が紐付くと、そのユーザーは登録待ちユーザーから外れます。また、患者さんのLINEには登録完了メッセージが送信されます。

登録完了メッセージの送信結果は、送信履歴に残ります。

患者区分では、足病変などの区分を選択できます。足病変の患者さんを登録する場合は、患者区分を「足病変」にし、メモ欄に部位やフォロー内容を記載しておくと管理しやすくなります。

患者一覧から患者を削除できます。削除すると患者マスタから外れますが、回答履歴と送信履歴は記録として残ります。

## 10. 足病変写真の受信

足病変の写真を患者さんからLINEで受け取ることができます。

事前準備:

- 患者登録で患者区分を「足病変」にする
- 設定画面で画像保存先Google DriveフォルダIDを登録する
- Driveフォルダをサービスアカウントメールアドレスへ編集者共有する
- Google CloudでGoogle Drive APIを有効化する

患者さんがLINEで画像を送ると、MedSafety Linkは画像を受信し、Google Driveフォルダへ保存します。回答履歴とダッシュボードには「画像を受信しました」と表示され、「画像を開く」リンクから確認できます。

画像は医療情報を含む可能性があります。保存先Driveフォルダの共有範囲は、必要な医療者のみに限定してください。

### 10-1. 位置情報の受信

患者さんがLINEで位置情報を送ると、MedSafety Linkは回答履歴に記録します。

記録される主な項目:

- 回答時刻
- 患者ID
- 氏名
- LINE user_id
- 種別
- 回答内容
- 重症度
- 緯度
- 経度
- 地図URL

回答履歴とダッシュボードには「地図を開く」リンクが表示されます。

### 10-2. Looker Studioでの可視化

Google Sheetsの `responses` タブをLooker Studioのデータソースとして接続すると、安否回答、重症度、位置情報を可視化できます。

Looker Studioで使いやすい列:

- `timestamp`: 回答時刻
- `patient_id`: 患者ID
- `name`: 氏名
- `event_type`: 通常モードまたは災害モード
- `code`: 回答コード
- `label`: 回答内容
- `handled`: 対応済みかどうか
- `severity`: 重症度
- `severity_score`: 重症度スコア
- `latitude`: 緯度
- `longitude`: 経度

作成しやすいグラフ:

- 地図: `latitude` と `longitude` を使って患者さんから送られた位置情報を表示
- 表: 未対応かつ重症度が高い回答を一覧表示
- スコアカード: 未対応数、重症アラート数、回答数を表示
- 時系列グラフ: 回答数や重症度の推移を表示

## 11. ダッシュボード

ダッシュボードでは、現在の運用状況を確認できます。

表示内容:

- 施設ID、施設名、現在モード
- 患者数
- LINE登録済み数
- 回答済み数
- 未回答数
- 緊急未対応数
- 登録待ち数
- 重症アラート
- 重傷者リスト
- 最新回答状況
- 直近の回答履歴

最新回答状況は回答時刻順で確認できます。

管理画面を開いている間は、1分ごとに新着回答を確認します。通知音を有効化している場合、新着回答があると音が鳴ります。

## 12. 患者一覧

患者一覧では、患者ごとの最新回答状況を確認できます。

主な操作:

- 最新回答の確認
- 対応済みにする
- 対応済みを解除する
- 個別メッセージを送る

緊急度が高い回答は色付きで表示されます。

## 13. 回答履歴

回答履歴では、これまでに届いた回答を時系列で確認できます。時刻は日本時間で表示されます。

確認できる項目:

- 時刻
- 患者ID
- 氏名
- LINE user_id
- 種別
- 回答内容
- 対応状況

回答履歴からも、対応済み操作と個別送信ができます。

## 14. 送信履歴

送信履歴では、MedSafety Linkから患者さんへ送信した内容を確認できます。

記録される主な送信:

- 一斉送信
- 未回答再送
- 個別送信
- LINE紐付け完了メッセージ

確認できる項目:

- 時刻
- 患者ID
- 氏名
- LINE user_id
- 種別
- 送信内容
- 結果
- 詳細

## 15. 一斉送信

一斉送信では、LINE登録済み患者さんへ回答ボタン付きの安否確認メッセージを送信します。

手順:

1. 「一斉送信」を開きます。
2. メッセージを確認または編集します。
3. 「登録済み患者に回答ボタン付きで一斉送信」を押します。

送信されるボタン名と回答コードは、設定画面の「安否確認ボタン・回答コード」で変更できます。

一斉送信の文面は、施設ごとに設定できます。

## 16. 未回答再送

未回答再送では、まだ回答していない患者さんだけに再送できます。

手順:

1. 「未回答再送」を開きます。
2. 再送メッセージを確認または編集します。
3. 送信します。

Googleスプレッドシートの最新回答状況をもとに未回答者を判定します。

## 17. 個別送信

個別送信は、患者一覧または回答履歴から実行できます。

設定画面で登録した「個別送信の定型文3件」を選択できます。必要に応じて、送信前に文面を編集してください。

送信結果は送信履歴に残ります。

## 18. モード切替

モードは2種類あります。

| モード | 動作 |
| --- | --- |
| 通常 | 診療時間内は通常対応。時間外、土日、祝日は時間外メッセージを返信 |
| 災害 | 24時間、災害対応として回答を受け付け |

通常モードで時間外に患者さんからメッセージが届いた場合、次の内容の時間外メッセージを返信します。

```text
ご連絡ありがとうございます。現在は時間外のため、すぐに対応ができません。診療時間内に順次確認いたします。なお、緊急の場合は当院までお電話をして頂いたうえで、救急外来受診をご検討ください。
```

設定画面で電話番号を登録している場合、時間外メッセージ内に連絡先として表示できます。

災害時は「モード切替」で災害モードにしてから、一斉送信を行います。

## 19. 通知音

管理画面では新着回答を1分ごとに確認し、新着があれば通知音を鳴らせます。

使い方:

1. 管理画面を開きます。
2. 上部の「通知音を有効化」を押します。
3. テスト音が鳴ることを確認します。

音を止めたい場合は「通知音をオフ」を押します。

音が鳴らない場合:

- ブラウザで `medsafety-link.onrender.com` がミュートになっていないか確認します。
- パソコン本体の音量を確認します。
- ブラウザの自動再生制限により、最初にボタン操作が必要な場合があります。
- タブが完全に閉じている場合、ブラウザ上の通知音は鳴りません。

## 20. 施設ID管理

施設ID管理では、登録済み施設の追加、操作対象の切替、削除ができます。

表示されるのは、システム管理者として許可された施設IDでログインした場合のみです。現在の運用では、`kumamoto_chuo` でログインした場合に表示されます。

主な操作:

- 新しい施設IDを追加
- 登録済み施設を確認
- この施設を操作
- 施設を削除

施設を削除すると、その施設の設定はMedSafety Linkの設定ファイルから削除されます。Googleスプレッドシート自体は自動削除されません。

## 21. 複数施設運用の注意

同じブラウザで複数施設をタブ表示する場合、URLの `active_institution_id` によって操作対象を区別します。

確認すべき点:

- 画面上部の施設IDが正しい
- 設定画面のLINEトークンが対象施設のもの
- スプレッドシートIDが対象施設のもの
- 一斉送信前に、現在操作中の施設名を確認する

施設ごとにLINE公式アカウントとGoogleスプレッドシートを分ける運用を推奨します。

## 22. Render運用と保存先

Render版では、設定保存先として永続ディスクを使用します。

推奨設定:

```text
SETTINGS_PATH=/var/data/settings.json
Disk mountPath=/var/data
```

設定保存先診断で次を確認します。

- 永続ディスク候補が存在: はい
- 永続ディスクを使用中: はい
- 登録施設数が想定通り

永続ディスクを使用していない場合、デプロイや再起動で追加施設や設定が消える可能性があります。

## 23. よくあるトラブル

### 23-1. LINEチャネルアクセストークンが未設定です

設定画面で、対象施設のチャネルアクセストークンを入力して保存してください。保存後、LINE接続診断を実行します。

### 23-2. LINE送信失敗: status=400

主な原因:

- LINE user_idが間違っている
- 患者さんが友だち追加していない
- 患者さんがLINE公式アカウントをブロックしている
- チャネルアクセストークンが別施設のもの
- Messaging APIチャネルが違う

### 23-3. Webhook検証はOKだが返信が届かない

確認項目:

- LINE DevelopersのWebhook利用がON
- Webhook URLが `https://medsafety-link.onrender.com/callback`
- LINE Official Account Managerの応答モードがBot
- 応答メッセージがOFF
- AI応答メッセージがOFF
- 設定画面のLINE destination / Bot userIdが保存されている

### 23-4. LINE公式アカウントの固定文が返る

LINE Official Account Managerの応答メッセージまたはAI応答メッセージがONの可能性があります。どちらもOFFにしてください。

### 23-5. Google Sheets APIが有効になっていない

Google Cloud Consoleで、対象プロジェクトのGoogle Sheets APIを有効化してください。

```text
https://console.cloud.google.com/apis/library/sheets.googleapis.com
```

### 23-6. Googleスプレッドシートへのアクセス権限がありません

スプレッドシート右上の「共有」から、サービスアカウントのメールアドレスを編集者として追加してください。

あわせて、設定画面のスプレッドシートIDが共有したシートのIDと一致しているか確認してください。

### 23-7. Invalid JWT Signature

サービスアカウントJSONの秘密鍵が無効、または古い可能性があります。

Google Cloudで新しいJSONキーを作成し、設定画面で再アップロードしてください。Renderの環境変数 `GOOGLE_SERVICE_JSON` に古い値がある場合は、削除または更新してください。

### 23-8. 患者ID、氏名、LINE IDの列がずれる

Googleスプレッドシートの見出し行が古い形式になっている可能性があります。

設定画面の「Googleシート初期化」を実行して、見出し行を整えてください。

### 23-9. 回答ボタンの文字と回答コードが合わない

設定画面の「安否確認ボタン・回答コード」を確認してください。

保存後、新しい一斉送信からテストしてください。過去回答は当時の内容のまま残ります。

### 23-10. 新しい施設が保存されない、デプロイ後に消える

Renderの永続ディスク設定を確認してください。

設定保存先診断で、`/var/data/settings.json` を使用しているか確認します。

### 23-11. Google接続診断は失敗するが共有済み

次を確認してください。

- スプレッドシートIDが正しい
- 共有した相手が、設定画面に表示されているサービスアカウントメールアドレスと一致している
- Google Sheets APIが有効
- JSONのGoogle Cloudプロジェクトが想定通り
- 古い `GOOGLE_SERVICE_JSON` 環境変数がRenderに残っていない

### 23-12. LINE画像がGoogle Driveへ保存されない

次を確認してください。

- 設定画面の画像保存先Google DriveフォルダIDが正しい
- Driveフォルダがサービスアカウントメールアドレスに編集者共有されている
- Google CloudでGoogle Drive APIが有効になっている
- サービスアカウントJSONが、Drive APIを有効化したGoogle Cloudプロジェクトのもの
- 患者さんのLINE user_idが患者登録で紐付いている

## 24. 日常運用の流れ

### 通常時

1. ダッシュボードを確認します。
2. 患者一覧で新着回答を確認します。
3. 必要な患者さんへ個別送信します。
4. 対応済みになった回答は対応状況を更新します。
5. 送信履歴、回答履歴を必要に応じて確認します。

### 災害時

1. モード切替で災害モードにします。
2. 一斉送信で安否確認を送ります。
3. ダッシュボードで回答状況を確認します。
4. 重症アラート、重傷者リストを優先して確認します。
5. 未回答再送を行います。
6. 個別対応が必要な患者さんへ個別送信します。
7. 対応済みになった回答を更新します。
