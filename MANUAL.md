# MedSafety Link 運用マニュアル

## 1. このシステムでできること

MedSafety Linkは、LINE公式アカウントとGoogle Sheetsを使って、災害時や通常診療時の患者安否確認を行う管理システムです。

主な機能は次の通りです。

- 施設ごとのログイン管理
- 患者登録とLINE user_idの紐付け
- 安否確認メッセージの一斉送信
- 未回答者への再送
- 個別メッセージ送信
- 回答一覧、回答履歴、ダッシュボード表示
- 通常モード、災害モードの切替
- 時間外メッセージの自動返信
- LINE接続、Webhook受信、Google Sheets接続の診断

## 2. 運用で使うURL

実運用ではRender版を使います。

```text
https://medsafety-link.onrender.com
```

LINE Developersに設定するWebhook URLは次です。

```text
https://medsafety-link.onrender.com/callback
```

パソコン内の `http://127.0.0.1:5001` は開発確認用です。端末を閉じるとローカル版は止まりますが、Render版は止まりません。

## 3. ログイン

1. `https://medsafety-link.onrender.com/login` を開きます。
2. 施設IDを入力します。
3. パスワードを入力します。
4. ログインすると管理画面に入ります。

施設IDとパスワードは、管理者が「施設ID管理」で作成します。

## 4. メニュー

管理画面の上部メニューは次の順番です。

- ダッシュボード
- 患者登録
- 回答一覧
- 回答履歴
- 一斉送信
- 未回答再送
- モード切替
- 設定
- Google連携手順
- 施設ID管理
- ログアウト

施設ID管理は、システム管理者として許可された施設だけに表示されます。

## 5. 初期設定の全体手順

新しい施設を使い始める時は、次の順に設定します。

1. 施設IDを作成します。
2. 施設IDでログインします。
3. 設定画面で施設情報を入力します。
4. LINEチャネルアクセストークンを登録します。
5. LINE接続診断を実行します。
6. GoogleサービスアカウントJSONをアップロードします。
7. GoogleスプレッドシートIDを登録します。
8. Googleスプレッドシートをサービスアカウントに共有します。
9. Google接続診断を実行します。
10. Googleシート初期化を実行します。
11. 患者登録を行います。
12. LINE公式アカウントのWebhook設定を確認します。
13. テスト送信、テスト返信を行います。

## 6. 施設ID管理

施設ID管理では、複数施設を追加できます。

入力する項目は次の通りです。

- 施設ID
- 施設名
- 初期パスワード

施設IDはログイン時に使います。英数字、アンダーバーなど、分かりやすく重複しないIDを推奨します。

例:

```text
kumachu_HD
t1dm
hdkumachu
```

施設を追加した後は、その施設IDでログインし、施設ごとのLINE設定とGoogle設定を行います。

## 7. 設定画面

設定画面では、ログイン中の施設だけに適用される内容を保存します。

### 施設情報

施設名、診療科、電話番号、ログインパスワードを設定します。

電話番号は、通常モードで時間外に患者からメッセージが届いた時の自動返信に表示されます。

### LINE設定

チャネルアクセストークンを入力します。

取得場所:

```text
LINE Developers
対象チャネル
Messaging API
チャネルアクセストークン
```

保存後、「LINE接続診断」を押します。成功すると、LINE destination / Bot userIdが自動で保存されます。

### 管理者LINE ID

緊急回答が届いた時に通知を受け取る管理者のLINE user_idを入力します。

複数人に通知する場合は、カンマ区切りで入力します。

```text
Uxxxxxxx,Uyyyyyyy
```

管理者本人がLINE公式アカウントへメッセージを送ると、未登録ユーザーまたは候補として確認できます。

### メッセージ設定

施設ごとに標準文面を設定できます。

- 一斉送信の標準メッセージ
- 未回答再送の標準メッセージ
- 個別送信の標準メッセージ

一斉送信画面や未回答再送画面で、送信前に文面を修正することもできます。

### 安否確認ボタン・回答コード

LINEに表示されるボタン名と、回答履歴に保存されるコードを設定します。

標準設定は次の通りです。

| ボタン | 表示名 | 回答コード |
| --- | --- | --- |
| 1 | 無事 | SAFE |
| 2 | 体調不良 | SICK |
| 3 | 薬・インスリン不足 | INSULIN_OUT |
| 4 | 低血糖が心配 | HYPO |
| 5 | 至急連絡希望 | CALL |

LINE内部では `1` から `5` の値で送受信し、システム側で施設設定に合わせて表示名と回答コードに変換します。

## 8. Google Sheets連携

### 8-1. Google CloudでサービスアカウントJSONを作成する

1. Google Cloud Consoleを開きます。

```text
https://console.cloud.google.com/
```

2. 使用するプロジェクトを選択します。
3. 「IAMと管理」から「サービスアカウント」を開きます。

```text
https://console.cloud.google.com/iam-admin/serviceaccounts
```

4. サービスアカウントを作成します。
5. 作成したサービスアカウントを開き、「キー」からJSONキーを作成します。
6. JSONファイルをダウンロードします。

### 8-2. Google Sheets APIを有効化する

Google Sheets APIが無効だと接続できません。

```text
https://console.cloud.google.com/apis/library/sheets.googleapis.com
```

対象プロジェクトでGoogle Sheets APIを有効化してください。

### 8-3. JSONをMedSafety Linkへアップロードする

1. MedSafety Linkの「設定」を開きます。
2. 「サービスアカウントJSONファイル」でJSONを選択します。
3. 保存します。
4. 「スプレッドシート共有先メールアドレス」が表示されます。

### 8-4. スプレッドシートを共有する

1. 使用するGoogleスプレッドシートを開きます。
2. 右上の「共有」を押します。
3. MedSafety Linkに表示された共有先メールアドレスを追加します。
4. 権限を「編集者」にします。
5. 共有を完了します。

### 8-5. スプレッドシートIDを入力する

スプレッドシートURLの `/d/` と `/edit` の間の文字列がスプレッドシートIDです。

例:

```text
https://docs.google.com/spreadsheets/d/ここがスプレッドシートID/edit
```

このIDを設定画面の「施設専用スプレッドシートID」に入力し、保存します。

### 8-6. Google接続診断

設定画面の「Google接続診断」を押します。

成功すれば、Google Sheetsを読み書きできます。

失敗した場合は、表示されるエラー内容を確認します。

## 9. Googleシート初期化

Google接続が成功した後に、Googleシート初期化を行います。

初期化により、必要なシートと見出し行が作成または確認されます。

使用するシートは次の通りです。

- `patients`
- `pending_users`
- `responses`
- `system_mode`

すでにデータがある場合でも、通常は既存データを消さずに必要な見出しを整えます。

## 10. LINE Developersの設定

LINE Developersで、対象のMessaging APIチャネルを開きます。

確認する項目は次の通りです。

- Webhook URL: `https://medsafety-link.onrender.com/callback`
- Webhookの利用: ON
- Webhook検証: 成功
- チャネルアクセストークン: MedSafety Linkの設定画面に保存済み

LINE Official Account Manager側では、次を確認します。

- 応答モード: Bot
- Webhook: ON
- 応答メッセージ: OFF
- AI応答メッセージ: OFF

患者が返信した時に、LINE公式アカウント側の固定文が返る場合は、応答メッセージがONになっている可能性があります。

## 11. 患者登録

患者登録画面では、患者情報とLINE user_idを紐付けます。

患者がLINE公式アカウントへ何かメッセージを送ると、未登録ユーザーとして表示されます。

登録の流れ:

1. 患者にLINE公式アカウントを友だち追加してもらいます。
2. 患者に「テスト」などのメッセージを送ってもらいます。
3. 管理画面の「患者登録」を開きます。
4. 未登録ユーザーのLINE user_idを確認します。
5. 患者ID、氏名、電話番号などと紐付けます。
6. 保存します。

登録後、その患者には一斉送信や個別送信ができます。

## 12. ダッシュボード

ダッシュボードでは、運用状況を一覧で確認できます。

表示される主な内容:

- 患者数
- LINE登録済み数
- 回答済み数
- 未回答数
- 緊急未対応数
- 登録待ち数
- 最新回答状況
- 直近の回答履歴

管理画面を開いている間は、1分ごとに新着回答を確認します。

通知音を有効化している場合、新着回答があると通知音が鳴ります。ブラウザの仕様により、最初に「通知音を有効化」を押す必要があります。

## 13. 一斉送信

一斉送信では、LINE登録済み患者へ安否確認メッセージを送ります。

1. 「一斉送信」を開きます。
2. メッセージを確認または編集します。
3. 「登録済み患者に回答ボタン付きで一斉送信」を押します。

患者には、メッセージと回答ボタンが送信されます。

回答ボタンの表示名と回答コードは、設定画面の「安否確認ボタン・回答コード」で変更できます。

## 14. 未回答再送

未回答再送では、まだ回答していない患者だけに再送できます。

1. 「未回答再送」を開きます。
2. 再送メッセージを確認または編集します。
3. 送信します。

Google Sheetsの回答状況をもとに、未回答者へ送信します。

## 15. 回答一覧

回答一覧では、患者ごとの最新回答を確認できます。

緊急度が高い回答は、対応状況を確認しながら運用します。

必要に応じて、患者へ個別メッセージを送信できます。

## 16. 回答履歴

回答履歴では、これまでに届いた回答を時系列で確認できます。

時刻は日本時間で表示します。

確認する項目:

- 時刻
- 患者ID
- 氏名
- 種別
- 回答内容
- 対応状況

## 17. 個別送信

回答一覧から、患者へ個別にメッセージを送信できます。

施設設定の「個別送信の標準メッセージ」が初期文面として使われます。

## 18. モード切替

モードは2種類です。

| モード | 動作 |
| --- | --- |
| 通常 | 診療時間内は通常対応、時間外は時間外メッセージを返信 |
| 災害 | 24時間、災害対応として回答を受け付け |

通常モードで時間外、土日、祝日の場合、患者からのメッセージには時間外メッセージを返信します。

時間外メッセージには、設定画面の電話番号を表示できます。

## 19. 通知音

管理画面には通知音機能があります。

使い方:

1. 管理画面を開きます。
2. 「通知音を有効化」を押します。
3. テスト音が鳴ることを確認します。

音が鳴らない場合:

- ブラウザでサイトがミュートになっていないか確認します。
- パソコン本体の音量を確認します。
- SafariやChromeの自動再生制限により、先にボタン操作が必要な場合があります。
- 「通知音をオフ」になっていないか確認します。

## 20. 診断機能

設定画面には3つの診断ボタンがあります。

### LINE接続診断

LINEチャネルアクセストークンが正しいか確認します。

成功すると、LINE destination / Bot userIdを保存します。

### Webhook受信診断

LINEからMedSafety LinkへWebhookが届いているか確認します。

「まだWebhook受信記録がありません」と表示される場合は、患者または管理者がLINE公式アカウントへ「テスト」と送ってから再確認してください。

### Google接続診断

Google Sheetsへ接続できるか確認します。

診断結果には、使用中のサービスアカウント、プロジェクト、スプレッドシートIDなどが表示されます。

## 21. よくあるトラブル

### LINEチャネルアクセストークンが未設定です

設定画面で、対象施設のチャネルアクセストークンを入力して保存してください。

保存後、LINE接続診断を実行します。

### LINE送信失敗: status=400

主な原因:

- LINE user_idが間違っている
- 患者が友だち追加していない
- LINE公式アカウントがブロックされている
- チャネルアクセストークンが別施設のもの
- Messaging APIチャネルが違う

### Webhook検証はOKだが、患者返信が届かない

確認する項目:

- LINE DevelopersのWebhook利用がON
- Webhook URLが `https://medsafety-link.onrender.com/callback`
- LINE Official Account Managerの応答モードがBot
- 応答メッセージがOFF
- AI応答メッセージがOFF
- 設定画面のLINE destination / Bot userIdが保存されている

### LINE公式アカウントの固定文が返る

LINE Official Account Managerの応答メッセージがONの可能性があります。

応答メッセージとAI応答メッセージをOFFにしてください。

### Google Sheets APIが有効になっていない

Google Cloud Consoleで、対象プロジェクトのGoogle Sheets APIを有効化してください。

```text
https://console.cloud.google.com/apis/library/sheets.googleapis.com
```

### Googleスプレッドシートへのアクセス権限がありません

スプレッドシート右上の「共有」から、サービスアカウントのメールアドレスを編集者として追加してください。

また、設定画面のスプレッドシートIDが、共有したシートのIDと一致しているか確認してください。

### Invalid JWT Signature

サービスアカウントJSONの秘密鍵が無効、または古い可能性があります。

Google Cloudで新しいJSONキーを作成し、設定画面で再アップロードしてください。

Renderの環境変数 `GOOGLE_SERVICE_JSON` に古い値がある場合は、削除または更新してください。

### 患者ID、氏名、LINE IDの列がずれる

Google Sheetsの見出し行が古い形式になっている可能性があります。

Googleシート初期化を実行して、見出し行を整えてください。

### 回答ボタンの文字と回答コードが合わない

設定画面の「安否確認ボタン・回答コード」を確認してください。

ボタン表示名と回答コードを保存した後、新しい一斉送信からテストしてください。すでに保存済みの過去回答は、当時の内容のまま残ります。

## 22. 日常運用の流れ

通常時:

1. ダッシュボードを確認します。
2. 回答一覧で新着回答を確認します。
3. 必要な患者へ個別送信します。
4. 対応済みになった回答は対応状況を更新します。

災害時:

1. モード切替で災害モードにします。
2. 一斉送信で安否確認を送ります。
3. ダッシュボードで回答状況を確認します。
4. 未回答再送を行います。
5. 緊急回答を優先して対応します。
6. 対応状況を記録します。

## 23. 実運用前チェックリスト

- Render版URLでログインできる
- 各施設IDでログインできる
- 各施設のLINEチャネルアクセストークンが保存されている
- LINE接続診断が成功する
- Webhook URLが正しく設定されている
- Webhook受信診断で受信記録が確認できる
- GoogleサービスアカウントJSONが保存されている
- Google Sheets APIが有効
- スプレッドシートがサービスアカウントに編集者共有されている
- Google接続診断が成功する
- Googleシート初期化が完了している
- 患者登録ができる
- 一斉送信ができる
- 患者の回答が回答一覧、回答履歴、ダッシュボードに表示される
- 通常モードと災害モードの動作を確認済み
- 通知音が必要な端末で鳴る

## 24. 管理上の注意

- サービスアカウントJSONは外部に共有しないでください。
- LINEチャネルアクセストークンは外部に共有しないでください。
- 施設ごとにLINE公式アカウントとGoogleスプレッドシートを分ける運用を推奨します。
- パスワードは施設ごとに別のものを設定してください。
- 実運用前に、必ずテスト患者で送信、返信、回答表示まで確認してください。
